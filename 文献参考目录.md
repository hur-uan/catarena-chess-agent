## 8. Pi Agent 与 Agent 架构相关补充

“Pi Agent”在公开资料中存在不同指向。与本项目“搭建 agent 学习框架、参考智能体架构知识”最相关的资料主要分为两类：一类是 Physical Intelligence 的 `π0 / π0.5` 机器人基础模型，强调从感知、语言到动作的通用策略架构；另一类是 LLM Agent 架构资料，强调规划、记忆、工具调用、反思和多智能体协作。以下文献可作为本项目 agent 架构设计的补充参考。

49. Black, K., Brown, N., Driess, D., Esmail, A., Equi, M., Finn, C., Fusai, N., Groom, L., Hausman, K., Ichter, B., et al. (2024). *π0: A Vision-Language-Action Flow Model for General Robot Control*. arXiv:2410.24164.  
    链接：https://arxiv.org/abs/2410.24164  
    用途：Physical Intelligence 提出的 `π0` 模型。该工作将视觉、语言和动作统一到一个通用机器人控制模型中，强调“感知输入 -> 策略模型 -> 动作输出”的架构思想。虽然本项目是棋类 Agent 而非机器人控制，但其“统一策略模型”和“跨任务泛化”的设计思路可作为 agent 架构参考。

50. Physical Intelligence, Black, K., Brown, N., Darpinian, J., Dhabalia, K., Driess, D., Esmail, A., Equi, M., Finn, C., et al. (2025). *π0.5: a Vision-Language-Action Model with Open-World Generalization*. arXiv:2504.16054.  
    链接：https://arxiv.org/abs/2504.16054  
    用途：`π0.5` 是 `π0` 的扩展版本，重点讨论开放世界泛化和多源数据联合训练。适合用于说明 agent 系统如何通过多样化任务、语义预测和低层动作反馈提升泛化能力。

51. Physical Intelligence. *Physical Intelligence official site*.  
    链接：https://www.physicalintelligence.company/  
    用途：Physical Intelligence 的官方主页，可作为 `π / Pi` 系列机器人基础模型和公司研究方向的资料入口。

52. Knight, W. (2024). *This Is a Glimpse of the Future of AI Robot*. Wired.  
    链接：https://www.wired.com/story/physical-intelligence-home-robot  
    用途：介绍 Physical Intelligence 和 `π0` 模型的产业背景。该资料不是学术论文，但可用于补充说明 `π0` 作为通用机器人策略模型的现实应用场景。

53. Wang, X., Wang, C., Nekoei, H., Pal, C., Lacoste, A., Gella, S., Liu, B., & Taslakian, P. (2026). *Mem-π: Adaptive Memory through Learning When and What to Generate*. arXiv:2605.21463.  
    链接：https://arxiv.org/abs/2605.21463  
    用途：该工作关注 LLM Agent 的自适应记忆机制，即在合适时机生成对当前任务有用的指导信息。可用于支持本项目中 memory、历史记录和反馈信息进入下一轮决策的设计思想。

54. Yao, S., Zhao, J., Yu, D., Du, N., Shafran, I., Narasimhan, K., & Cao, Y. (2022). *ReAct: Synergizing Reasoning and Acting in Language Models*. arXiv:2210.03629.  
    链接：https://arxiv.org/abs/2210.03629  
    用途：ReAct 提出将推理轨迹和动作执行交替结合，是 LLM Agent 架构中的经典方法。可用于说明 agent 不只是一次性输出答案，而是通过“观察、思考、行动、再观察”的循环与环境交互。

55. Shinn, N., Cassano, F., Berman, E., Gopinath, A., Narasimhan, K., & Yao, S. (2023). *Reflexion: Language Agents with Verbal Reinforcement Learning*. arXiv:2303.11366.  
    链接：https://arxiv.org/abs/2303.11366  
    用途：Reflexion 使用语言形式的反思记忆帮助 Agent 在多轮任务中改进表现。本项目中“读取历史 memory、失败局面和上一轮结果，再进入下一轮优化”的结构与该思想相近。

56. Park, J. S., O'Brien, J. C., Cai, C. J., Morris, M. R., Liang, P., & Bernstein, M. S. (2023). *Generative Agents: Interactive Simulacra of Human Behavior*. arXiv:2304.03442.  
    链接：https://arxiv.org/abs/2304.03442  
    用途：该文提出包含记忆、反思和规划的生成式 Agent 架构。可作为报告中解释“Agent 需要长期记忆、经验抽象和计划生成能力”的架构参考。

57. Wang, G., Xie, Y., Jiang, Y., Mandlekar, A., Xiao, C., Zhu, Y., Fan, L., & Anandkumar, A. (2023). *Voyager: An Open-Ended Embodied Agent with Large Language Models*. arXiv:2305.16291.  
    链接：https://arxiv.org/abs/2305.16291  
    用途：Voyager 通过自动课程、技能库和环境反馈进行持续探索。虽然任务环境是 Minecraft，但其“反馈驱动改进、技能积累和自验证”的架构思想与本项目的轮间自学习框架相关。

58. Schick, T., Dwivedi-Yu, J., Dessì, R., Raileanu, R., Lomeli, M., Zettlemoyer, L., Cancedda, N., & Scialom, T. (2023). *Toolformer: Language Models Can Teach Themselves to Use Tools*. arXiv:2302.04761.  
    链接：https://arxiv.org/abs/2302.04761
