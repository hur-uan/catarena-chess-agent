# 项目说明

## 当前状态

项目当前是 `profile-only` 参数自学习系统，不是自动改代码系统。

正式实验语义已经收紧为：

- `pipeline_version`: `profile_only_v2_internal_runtime`
- `runtime_policy`: `formal_internal_only`
- `formal_execution_backend`: `internal_engine`

也就是说，正式实验默认只使用内部搜索引擎下棋，并只让具备成熟反馈闭环的参数进入在线自治优化。

## 运行时链路

运行时入口：

- `agents/chess_agent.py`

正式链路：

`observation -> strategy_profile -> search_router -> internal engine -> legal guard -> move`

保留的正式运行时工具：

- `tools/strategy_profile.py`
- `tools/search_router.py`
- `agents/engine.py`
- `tools/move_validator.py`

`black_numba` 仍可作为研究备用组件保留，但默认 `external_engine.enabled=false`。它不再属于正式默认工具链，因为当前在线调参只覆盖内部引擎参数。

## 自我提升链路

轮间入口：

- `optimization/meta_agent.py`

正式闭环：

`CATArena feedback -> failure classification -> feedback FEN suite -> SPSA -> paired self-play -> SPRT -> feedback prescreen -> historical regression -> promotion gate`

反馈输入仍来自 CATArena 可提供的信息：

- `battle.log`
- `ranking.csv`
- 历史 `memory.jsonl`
- 失败局面 FEN
- 每轮产物记录

## 当前正式在线参数面

当前唯一在线 `active` block：

- `search.history_order`

当前唯一正式可调参数：

- `search.tt_move_bonus`

已冻结参数：

- `search.history_bonus_scale`
- `search.history_bonus_power`

其他 block 只保留为离线研究注册项，不进入正式在线自治优化：

- `eval.weights`
- `piece_activity`
- `pawn_structure`
- `king_safety`
- `search.depth_time`
- `move_ordering.tactical`
- `move_ordering.development`
- `piece_values`
- `phase`
- `external_engine.*`

## 为什么现在能接正式实验

当前正式链路满足四个条件：

- 被调参数直接作用于正式下棋执行器。
- 参数搜索使用 `SPSA` 和成对自对弈。
- 接受候选使用 `SPRT`、feedback prescreen 和 promotion gate。
- 历史回归只比较同一运行时策略下产生的 profile。

因此现在的“自我提升”含义是：

- agent 根据 CATArena 反馈提出新的策略 profile。
- 候选 profile 经过本地反馈验证。
- 通过门控后才写回正式 profile。

它不再表示：

- LLM 自动重写棋手代码。
- 未验证参数直接上线。
- 外部工具链默认替代被调执行器。

## 运行方式

单轮：

```bash
env PYTHONPYCACHEPREFIX=.pycache .venv/bin/python \
  scripts/run_catarena_platform_round.py \
  --round-id formal_profile_001 \
  --backend profile \
  --reports-dir reports/formal_profile
```

连续多轮：

```bash
env PYTHONPYCACHEPREFIX=.pycache .venv/bin/python \
  scripts/run_continuous_learning.py \
  --start-round 1 \
  --round-count 10 \
  --round-prefix live_profile \
  --backend profile \
  --reports-dir reports/catarena_platform
```

每轮会生成：

- `optimization_report.json`
- `round_record.json`
- `evaluation_record.json`
- `parameter_record.json`
- `candidate_profile.json`
- `feedback_fen_suite.json`

这些文件用于复盘当前轮为什么调参、调了什么、是否通过门控、是否晋升。
